گیت وی FXO - FXS

 

در صورتیکه نیاز به افزودن خط شهری به مرکز تلفن ویپ باشد، می توان از دستگاه گیت وی ویپ آنالوگ استفاده نمود تا با اتصال خط شهری به پورت FXO، ارتباط با مرکز تلفن VOIP برقرار شود...

 

گیت وی E1-PRI

اگر خط E1 یا PRI دارید، برای افزودن آن به مرکز تلفن ویپ، می توانید از دستگاه گیت وی ویپ E1 یا PRI استفاده کنید تا با اتصال خط E1 به پورت E1 دستگاه گیت وی، ارتباط با مرکز تلفن VOIP برقرار شود...

 

گیت وی GSM

 

برای اتصال سیم کارت به مرکز تلفن ویپ، گیت وی ویپ جی اس ام راه حل مناسبی است. سیم کارت به راحتی روی گیت وی GSM نصب می شود و گیت وی ویپ جی اس ام، GSM را به VOIP تبدیل می کند.

  مرکز تلفن ویپ برای کسب و کارهای متوسط و بزرگ

مرکز تلفن VOIP، به صورت مستقل، ارتباط شبکه را با خط شهری و E1 برقرار مي‌کند و یک سیستم ویپ با امکانات پيشرفته را در شرکت یا سازمان شما فراهم می نماید. بدون احتیاج به یک سرور حجيم و پر دردسر و بدون نیاز به بررسی نرم افزارهای متعدد موجود در بازار، مرکز تلفن ویپ  مناسب برای دفاتر متوسط و بزرگ با ضريب اطمينان بالا راه اندازي نماييد.

 

مرکز تلفن ویپ برای کسب و کارهای کوچک

 

 

مرکز تلفن IP یا IPPBX امکان اتصال خطوط مخابرات را به داخلی های شما فراهم می کند. جهت راه اندازي يک مرکز تلفن ویپ  مطمئن و بي دردسر در یک شرکت کوچک، کافیست یکی از IPPBX های زیر را با ظرفيت دلخواه FXO یا FXS یا GSM انتخاب کنید تا بدون نیاز به سرور، کاملا مستقل و کارآمد، ارتباط دنياي VOIP با دنياي TDM را برقرار کنید و از امکانات فوق العاده VOIP در کسب و کار کوچک خود بهره مند شوید.

 

تلفن ویپ ساده

 

ساده ترین مدل های تلفن ویپ يا IP Phone، دارای یک یا دو SIP Account هستند که برای کاربرانی مناسبند که ترافیک تماس بالا ندارند و نیازی به مانیتور کردن وضعیت سایر داخلیها نیز ندارند.

 

تلفن ویپ پیشرفته


تلفن ویپ پیشرفته، بیش از دو SIP Account دارد و برای استفاده کاربرانی مناسب است که دارای ترافیک مکالمه بالا هستند و نیاز به مانیتور کردن وضعیت سایر داخلیها دارند. این تلفن دارای کلیدهای BLF جهت بررسی وضعیت داخلیها و شماره گیری سریع است.

 

تلفن ویپ تصویری

 

جهت برقراری ارتباط تصویری هنگام مکالمه تلفنی روی شبکه، می توانید از تلفن ویپ تصویری استفاده نمایید. برقراری یک تماس تلفنی همزمان با دریافت تصویر، علاوه بر موثر ساختن یک مکالمه، شما را از رفت و آمدهای زمان بر بی نیاز می کند.

تلفن ویپ بی سیم

تلفن ویپ بی سیم، با استفاده از تکنولوژی DECT، امکان جابه جایی تا 300 متر را برای کاربر فراهم می کند. کاربرانی که به دلیل نوع فعالیت در سازمان، در طول ساعات کاری در یک مکان ثابت یا پشت میز کار خود حضور ندارند، راهکار استفاده از تلفن ویپ بی سیم را انتخاب می کنند.

اتصال به گیت وی:

اتصال به گیتوی از سه طریق زیر امکان پذیر است

1- کنسول

2- telnet

3- رابط وب

کنسول:

برای اتصال به پورت کنسول کابل سریال خود را به پورت کنسول دستگاه متصل کرده و تنظیمات زیر را با استفاده از برنامه  HyperTerminal انجام دهید.

Speed = 115200

Data = 8 bits

Parity = None

Stop = 1 bit

username و password پیش فرض admin می‌باشد.

برای اتصال از طریق telnet یا رابط وب نیاز به داشتن IP دستگاه است. گیتوی‌های سنگوما به صورت پیش فرض بر روی هر دو پورت lan خود، از dhcp تلاش می‌کند تاip دریافت کند. اگر در شبکه خود dhcp سرور ندارید می‌توانید با استفاده از کابل کنسول به گیتوی متصل شده و به صورت دستی ip را وارد نمایید.

set lan.if.1.use_dhcp=0

set lan.if.1.ip=192.168.1.6

set lan.if.1.subnet=255.255.255.0

save

reboot system

بعد از ریبوت شدن دستگاه ip خود را بر روی browser وارد کنید

تنظیمات گیتوی:

Dialplan:

برای هدایت تماس‌های ورودی و خروجی از روی پورت‌های مختلف باید با تعریف کردن Dialplan برای پورت مربوطه، آن را مشخص کنید.

 

به طور کلی برای ایجاد یک Dialplan نیاز به مشخص کردن تماس‌های ورودی و خروحی می‌باشد

برای مثال: تماس‌های ورودی از روی پورت‌های fxo از ترانک SIP خارج شوند

 

ارسال از روی ترانک SIP:

برای ارسال از روی ترانک ابتدا به قسمت Dialplan بروید. به صورت پیش فرض یک پروفایل برای ارسال از روی ترانک SIP با نام To_SIP وجود دارد.

 

 

برای مشخص کردن تماس‌های ورودی و خروجی می‌توانید از دو پارامتر Source و Destination استفاده کنید.

 

 

 

مشخص کردن تماس ورودی:

برای مشخص کردن interface ورودی می‌توانید از پارامتر IF به همراه شماره interface مورد نظر استفاده کنید.

نکته: هر پورت در گیتوی‌های سنگوما دارای شماره interface می‌باشند.

برای دیدن شماره interface بر روی پورت‌های FXS و FXO به قسمت Quick config بروید. (همانند شکل زیر)

 

 

 

 

 

نکته: به صورت پیش فرض پورت‌های FXS از رنج 0101 شروع می‌شوند و پورت‌های FXO از رنج 0201.

 

حال با مشخص شدن شماره interface می‌توانید آن را در پارامتر IF وارد کنید. برای تعریف interface شما می‌توانید با استفاده regular expression، آن را محدود کنید.

به مثال‌های زیر توجه کنید.

IF:0201                          فقط اینترفیس شماره 0201

IF:0[2]..                          اینترفیس‌های شماره 0200 تا 0299

IF:0[2]0[1-4]                  اینترفیس‌های شماره 0201 تا 0204

IF:0[12]..                        اینترقیس‌های شماره 0100 تا 0199 و 0200 تا 0299

مشخص کردن تماس‌های خروجی:

برای مشخص کردن مسیر خروجی نیز همانند بالا نیاز به interface خروجی و چند پارامتر اختیاری دیگر داریم.

شما می‌توانید از پارامتر‌های TA و TEL نیز برای مشخص کردن آدرس network و همین طور شماره تماس گیری بر روی آن کانال استفاده کنید.

به مثال زیر توجه کنید.

Source:

IF:01..,TEL:<.*>

Destination:

IF:9901,TEL:<1>,TA:192.168.1.16
در مثال بالا تمامی تماس‌های دریافتی از روی پورت‌های FXS به سمت ترانک SIP با آدرس 192.168.1.16 ارسال خواهد شد.

در قسمت Source پارامتر TEL برای محدود کردن شماره‌های ارسالی توسط کاربر می‌باشد. در مثال بالا <*.>:TEL در واقع تمامی اعداد دریافت شده را قبول می‌کند.

حال به مثال زیر توجه کنید:

Source:

IF:01..,TEL:<9><.*>

Destination:

IF:9901,TEL:<2>,TA:192.168.1.16

در مثال بالا ما در واقع ۲ دنباله در قسمت Source تعریف کرده‌ایم. اولی برای محدود کردن شماره‌ کاربران با عدد ۹ و دومی برای شماره ارسالی. در قسمت Destination ما قسمت دوم دنباله را بر روی ترانک SIP خود می‌فرستیم.

 

نکته: در تمامی مثال‌های بالا interface 9901 نشان‌گر ترانک SIP می‌باشد
